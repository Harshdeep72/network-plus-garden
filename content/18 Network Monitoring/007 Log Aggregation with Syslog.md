## 📖 Core Concept
Network logging is the process of collecting data regarding events, warnings, alerts, and critical information generated by network devices. In small environments, manual review is possible; however, in enterprise environments with hundreds or thousands of devices, manual review is impossible.

**Log Aggregation** is the practice of collecting logs from disparate devices into a centralized location. This allows administrators to:
* **Normalize and Correlate:** Combine data from different sources to identify patterns or trends.
* **Baseline Analysis:** Compare current traffic/events against "normal" behavior to detect anomalies (e.g., a sudden spike in data transfer or unauthorized port usage).
* **Incident Investigation:** Reconstruct events after a security breach or system failure.

## 🛠️ Hardware & Devices
* **Network Devices:** Routers, switches, firewalls, and servers (acting as log clients).
* **Syslog Server/SIEM:** A centralized server (or appliance) that receives, stores, and analyzes log data from multiple clients.

## 📜 Protocols & Standards
* **Syslog:** A standard protocol used to send log messages to a centralized server.
 * **Transport Protocol:** UDP (User Datagram Protocol).
 * **Port Number:** 514.

## 🔑 Key Terminology
* **Syslog:** A protocol for message logging that allows separation of the software that generates messages, the system that stores them, and the software that reports and analyzes them.
* **SIM (Security Information Management):** A system focused on the collection and long-term storage of security-related logs.
* **SEM (Security Event Management):** A system focused on real-time monitoring, correlation of events, and notifications.
* **SIEM (Security Information and Event Management):** A modern, comprehensive system that combines the functionality of SIM and SEM into a single platform.
* **Client:** Any device (router, switch, firewall, server) configured to send its log data to a centralized server.
* **Server:** The centralized repository that receives and stores logs from various clients.
* **Network Traffic Logs:** Records of traffic flows (source/destination IP, ports, MAC addresses, packet length, TTL) used to identify anomalies.
* **Audit Log (Audit Trail):** A record of events and changes made to a device, such as configuration changes, to track who did what and when.
* **Application Log (Windows):** Logs specific to software applications (e.g., Microsoft Word crashes).
* **Security Log (Windows):** Logs related to security events, such as successful or failed login attempts.
* **System Log (Windows):** Logs related to the operating system itself (e.g., driver issues, OS errors).

## 🔄 How It Works (Processes)
1. **Configuration:** The administrator configures network devices (clients) to point to the IP address of the Syslog/SIEM server.
2. **Severity Filtering:** The administrator defines which severity levels (0–7) should be sent to the server to manage storage space and reduce noise.
3. **Transmission:** The client sends log data via UDP port 514 to the server.
4. **Aggregation:** The server receives logs from multiple sources and stores them in a centralized database.
5. **Analysis:** Administrators use the server interface to search, filter, and correlate logs to identify trends or security incidents.

### Syslog Severity Levels
| Level| Severity| Description|
| :---| :---| :---|
| 0| Emergency| System is unusable/unstable.|
| 1| Alert| Action must be taken immediately.|
| 2| Critical| Failure in a primary application; immediate attention required.|
| 3| Error| System is unable to function properly (e.g., storage full).|
| 4| Warning| Potential error; action should be taken soon.|
| 5| Notice| Unusual events that are not errors.|
| 6| Informational| Normal operational messages (e.g., app started/stopped).|
| 7| Debugging| Information useful for developers.|

## 📊 Comparisons & Key Differences
* **Syslog vs. SIEM:** Syslog is the *protocol* used to transport logs; a SIEM is the *platform* that receives, stores, and performs advanced correlation/analysis on those logs.
* **Windows Log Types:**
 * **Application:** Software-specific issues.
 * **Security:** Authentication and access events.
 * **System:** OS-level events.
* **Windows Severity Indicators:**
 * **Red Exclamation:** Error.
 * **Yellow Triangle:** Warning.
 * **White Circle (Blue 'i'):** Informational.

## ⚠️ Exam Tips & Important Notes
* **Memorize the Severity Levels:** You may be asked to identify the specific level number for a given condition (e.g., "What is level 3?"). Remember: 0 is most severe, 7 is least severe.
* **Syslog Protocol:** Remember it uses **UDP port 514**.
* **Baseline Knowledge:** You cannot identify an anomaly (like a 10GB data spike) if you do not know what "normal" looks like for your network.
* **Storage Management:** Administrators must balance the need for logs against available disk space by choosing which severity levels to log.

## 🔑 Key Terminology
- **Term 1**: session layers keeps conversations separate to prevent intermingling of data

## 🔄 How It Works (Processes)
- setup session : checking user credentials and assigning numbers to session to help identify them.
- Maintain Session where data transfers back and forth across the network.
- transfer data after session is established if there is a break in connection we reestablish connection and get acknowledge receipt making sure data was sent back and forth .
- Tear Down session : ending of a session after the transfer is done or when the other party disconnects. or we can tear down connection mutually .
